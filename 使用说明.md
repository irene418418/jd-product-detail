# 京东商品详情页 - 使用说明

## 项目概述

这是一个完整的京东商品详情页实现，包含了电商系统中商品详情页的所有核心功能。

## 安装和运行

### 1. 安装依赖

```bash
cd jd-product-detail
npm install
```

### 2. 启动开发服务器

```bash
npm run dev
```

服务器将在 http://localhost:3000 启动

### 3. 访问页面

在浏览器中访问：
- 首页：http://localhost:3000
- 商品详情页：http://localhost:3000/product/123

## 功能说明

### 1. 商品信息展示

- **商品标题**：显示完整的商品名称
- **品牌信息**：展示商品品牌
- **评分和评价**：显示商品评分（星级）和评价数量
- **商品标签**：展示热销、新品等标签

### 2. 图片浏览

- **主图展示**：大图显示当前选中的商品图片
- **缩略图列表**：点击缩略图可切换主图
- **大图预览**：点击主图可打开全屏预览模式
- **加载状态**：图片加载时显示加载动画

### 3. 规格选择

- **多规格支持**：支持颜色、尺寸等多种规格
- **规格联动**：选择一个规格后，其他规格会自动计算可用状态
- **库存提示**：无库存的规格会自动禁用并显示删除线
- **SKU匹配**：选择完整规格后自动匹配对应的SKU

### 4. 购买操作

- **价格显示**：显示当前价格和原价（如有折扣）
- **库存提示**：显示当前库存数量，库存不足时会特别提示
- **数量选择**：可以增减购买数量，自动限制在有效范围内
- **加入购物车**：将商品加入购物车
- **立即购买**：直接跳转到结算页面

### 5. 商品详情

- **商品描述**：支持富文本HTML内容展示（已做XSS防护）
- **规格参数**：以表格形式展示商品的详细参数
- **售后服务**：列出所有售后服务项目
- **标签页切换**：可在不同内容区域间切换

### 6. 响应式设计

- **桌面端**（>1200px）：三列布局，信息展示完整
- **平板端**（768px-1200px）：两列布局，购买面板独占一行
- **移动端**（<768px）：单列布局，适合手机浏览

## 业务逻辑说明

### 规格联动算法

当用户选择某个规格时，系统会：
1. 遍历所有SKU列表
2. 找出与当前选择匹配且有库存的SKU
3. 提取这些SKU中其他规格的可选值
4. 将不在可选值中的规格标记为禁用

### 数量验证

购买数量会自动验证：
- **最小值**：1件
- **最大值**：取库存和最大购买限制（99件）的较小值
- **非法输入**：自动修正为最小值

### 购买前验证

点击"加入购物车"或"立即购买"时会依次检查：
1. **登录状态**：未登录会跳转到登录页
2. **规格选择**：未选择完整规格会提示
3. **库存检查**：库存不足会提示

## 技术架构

### 状态管理

使用Redux Toolkit管理全局状态：
- **productSlice**：商品数据、规格选择、SKU匹配
- **cartSlice**：购物车状态
- **userSlice**：用户登录状态

### 组件结构

```
ProductDetailPage (页面容器)
├── ImageGallery (图片浏览)
├── ProductInfo (商品信息)
├── SpecSelector (规格选择器)
├── PurchasePanel (购买面板)
└── ProductDescription (商品详情)
```

### API服务

- **ProductService**：商品相关API
- **CartService**：购物车相关API
- **UserService**：用户相关API

## 开发说明

### 接入真实API

当前使用模拟数据。要接入真实API：

1. 修改 `src/services/apiClient.ts` 中的 `baseURL`：
```typescript
baseURL: 'https://your-api-domain.com/api'
```

2. 更新 `src/store/slices/productSlice.ts` 中的 `fetchProductDetail`：
```typescript
const response = await ProductService.getProductDetail(productId);
return response;
```

3. 更新 `src/store/slices/cartSlice.ts` 中的 `addToCart`：
```typescript
const response = await CartService.addToCart(request);
return response;
```

### 添加新功能

1. **添加新组件**：在 `src/components/` 目录下创建
2. **添加新页面**：在 `src/pages/` 目录下创建
3. **添加新状态**：在 `src/store/slices/` 目录下创建新的slice
4. **添加新API**：在 `src/services/` 目录下创建新的service

### 样式定制

所有组件使用CSS Modules，样式文件命名为 `index.module.less`。

修改样式：
1. 找到对应组件的样式文件
2. 修改或添加样式规则
3. 保存后自动热更新

### 性能优化建议

1. **图片优化**：使用CDN加速，启用懒加载
2. **代码分割**：使用React.lazy动态导入大组件
3. **缓存策略**：合理使用useMemo和useCallback
4. **虚拟滚动**：长列表使用虚拟滚动技术

## 常见问题

### Q: 如何修改默认端口？

A: 修改 `vite.config.ts` 中的 `server.port` 配置。

### Q: 如何添加新的规格类型？

A: 规格类型由后端API返回，前端会自动渲染。确保API返回的数据符合 `SpecGroup` 接口定义。

### Q: 如何自定义错误提示？

A: 修改 `src/utils/errorHandler.ts` 中的 `errorMessages` 映射表。

### Q: 如何集成真实的登录系统？

A: 实现 `src/services/userService.ts` 中的登录相关方法，并更新 `src/hooks/useAuthGuard.ts` 中的登录跳转逻辑。
