# 部署指南

本文档介绍如何将项目部署到 GitHub Pages。

## 前置准备

1. 确保你已经将项目推送到 GitHub 仓库
2. 确保你有仓库的管理权限

## 自动部署（推荐）

项目已配置 GitHub Actions 自动部署，每次推送到 `main` 分支时会自动构建并部署。

### 启用 GitHub Pages

1. 进入你的 GitHub 仓库
2. 点击 `Settings`（设置）
3. 在左侧菜单找到 `Pages`
4. 在 `Source` 下选择 `GitHub Actions`
5. 保存设置

### 触发部署

推送代码到 `main` 分支：

```bash
git add .
git commit -m "部署到 GitHub Pages"
git push origin main
```

部署完成后，你可以通过以下地址访问：
```
https://你的用户名.github.io/jd-product-detail/
```

### 查看部署状态

1. 进入仓库的 `Actions` 标签页
2. 查看最新的工作流运行状态
3. 如果部署失败，点击查看详细日志

## 手动部署

如果需要手动部署，可以按照以下步骤：

### 1. 构建项目

```bash
npm run build
```

### 2. 部署到 GitHub Pages

使用 `gh-pages` 工具：

```bash
# 安装 gh-pages
npm install -D gh-pages

# 添加部署脚本到 package.json
# "deploy": "gh-pages -d dist"

# 执行部署
npm run deploy
```

## 配置说明

### Base URL 配置

项目的 `vite.config.ts` 已配置 base 路径：

```typescript
base: process.env.NODE_ENV === 'production' ? '/jd-product-detail/' : '/'
```

如果你的仓库名不是 `jd-product-detail`，请修改为你的仓库名。

### 自定义域名（可选）

如果你想使用自定义域名：

1. 在 `public` 目录下创建 `CNAME` 文件
2. 文件内容为你的域名，例如：`www.example.com`
3. 在域名提供商处配置 DNS 记录指向 GitHub Pages

## 更新配置

部署前，请确保更新以下文件中的占位符：

### README.md
- 将 `你的用户名` 替换为你的 GitHub 用户名
- 更新在线演示链接

### package.json
- 更新 `repository.url`
- 更新 `bugs.url`
- 更新 `homepage`
- 填写 `author` 信息

### vite.config.ts
- 确认 `base` 路径与仓库名一致

## 常见问题

### 1. 页面显示 404

检查 `vite.config.ts` 中的 `base` 配置是否正确。

### 2. 资源加载失败

确保所有资源路径使用相对路径或正确的 base 路径。

### 3. 路由不工作

GitHub Pages 不支持 SPA 的客户端路由，需要配置：
- 使用 Hash 路由
- 或添加 404.html 重定向配置

### 4. 部署失败

检查 GitHub Actions 日志：
- 确认 Node.js 版本兼容
- 确认依赖安装成功
- 确认构建命令正确

## 环境变量

如果项目使用环境变量，需要在 GitHub 仓库设置中配置：

1. 进入 `Settings` > `Secrets and variables` > `Actions`
2. 添加需要的环境变量
3. 在 `.github/workflows/deploy.yml` 中引用

## 回滚部署

如果需要回滚到之前的版本：

1. 进入 `Actions` 标签页
2. 找到之前成功的部署
3. 点击 `Re-run jobs` 重新运行

或者：

```bash
# 回退到指定提交
git reset --hard <commit-hash>
git push -f origin main
```

## 监控和维护

- 定期检查 GitHub Actions 运行状态
- 关注依赖更新和安全警告
- 监控网站访问情况

---

如有问题，请查看 [GitHub Pages 文档](https://docs.github.com/pages) 或提交 Issue。
