# 京东商品详情页 - 启动指南

## 📋 前置要求

在开始之前，请确保您的开发环境满足以下要求：

### 必需软件

- **Node.js**: 版本 16.0 或更高（推荐使用 18.x 或 20.x LTS 版本）
- **npm**: 版本 7.0 或更高（通常随 Node.js 一起安装）
- **现代浏览器**: Chrome、Firefox、Edge 或 Safari 最新版本

### 检查当前版本

打开终端（命令行），运行以下命令检查版本：

```bash
node --version
npm --version
```

如果未安装 Node.js，请访问 [Node.js 官网](https://nodejs.org/) 下载安装。

---

## 🚀 安装步骤

### 1. 进入项目目录

```bash
cd jd-product-detail
```

### 2. 安装项目依赖

运行以下命令安装所有必需的依赖包：

```bash
npm install
```

**说明**：
- 首次安装可能需要 2-5 分钟，具体时间取决于网络速度
- 如果安装速度较慢，可以考虑使用国内镜像源（淘宝镜像）：
  ```bash
  npm install --registry=https://registry.npmmirror.com
  ```

### 3. 验证安装

安装完成后，检查 `node_modules` 目录是否已创建，该目录包含所有依赖包。

---

## 🎯 启动开发服务器

### 启动命令

在项目根目录下运行：

```bash
npm run dev
```

### 预期输出

启动成功后，您将看到类似以下的输出信息：

```
  VITE v5.3.4  ready in 500 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
  ➜  press h + enter to show help
```

**重要提示**：
- 默认端口是 `5173`，如果该端口被占用，Vite 会自动使用下一个可用端口（如 5174、5175 等）
- 开发服务器启动后会保持运行状态，不要关闭终端窗口

## 🔧 常见问题排查

### 问题 1: 页面显示空白

**可能原因**：
- JavaScript 错误导致页面渲染失败
- 路由配置问题
- 组件加载失败

**解决方法**：
1. 打开浏览器开发者工具（按 F12）
2. 查看 Console（控制台）标签页中的错误信息
3. 查看 Network（网络）标签页，确认资源是否正常加载
4. 检查终端中是否有编译错误信息

### 问题 2: 无法连接到开发服务器

**症状**：浏览器显示"无法访问此网站"或"连接被拒绝"

**解决方法**：
1. 确认开发服务器是否正在运行（检查终端窗口）
2. 检查端口号是否正确（查看终端输出的实际端口）
3. 尝试使用 `http://127.0.0.1:5173` 代替 `localhost`
4. 检查防火墙设置，确保允许访问该端口
5. 重启开发服务器：
   ```bash
   # 按 Ctrl+C 停止服务器
   # 然后重新运行
   npm run dev
   ```

### 问题 3: 依赖安装失败

**症状**：运行 `npm install` 时出现错误

**解决方法**：
1. 清除 npm 缓存：
   ```bash
   npm cache clean --force
   ```
2. 删除 `node_modules` 目录和 `package-lock.json` 文件：
   ```bash
   rm -rf node_modules package-lock.json
   ```
3. 重新安装依赖：
   ```bash
   npm install
   ```
4. 如果仍然失败，尝试使用淘宝镜像源

### 问题 4: 端口被占用

**症状**：启动时提示端口已被使用

**解决方法**：
1. 方法一：Vite 会自动使用下一个可用端口，查看终端输出的实际端口号
2. 方法二：手动指定端口：
   ```bash
   npm run dev -- --port 3000
   ```
3. 方法三：找到并关闭占用端口的进程：
   - Windows: `netstat -ano | findstr :5173`
   - Mac/Linux: `lsof -i :5173`

### 问题 5: 热更新不生效

**症状**：修改代码后页面不自动刷新

**解决方法**：
1. 手动刷新浏览器（F5 或 Ctrl+R）
2. 检查文件是否正确保存
3. 重启开发服务器
4. 清除浏览器缓存（Ctrl+Shift+Delete）

### 问题 6: TypeScript 编译错误

**症状**：终端显示类型错误

**解决方法**：
1. 检查错误信息，定位到具体文件和行号
2. 确保所有类型定义正确
3. 如果是第三方库的类型问题，尝试安装对应的 `@types` 包
4. 临时跳过类型检查（不推荐）：修改 `tsconfig.json` 中的 `strict` 选项

## 🛠️ Chrome 开发者工具使用方法

### 打开开发者工具

- **方法 1**: 按 `F12` 键
- **方法 2**: 按 `Ctrl+Shift+I`（Windows/Linux）或 `Cmd+Option+I`（Mac）
- **方法 3**: 右键点击页面，选择"检查"或"审查元素"

### 主要功能标签页

#### 1. Console（控制台）
- **用途**: 查看 JavaScript 错误、警告和日志信息
- **常用操作**:
  - 查看红色错误信息（Error）
  - 查看黄色警告信息（Warning）
  - 使用 `console.log()` 输出的调试信息

#### 2. Network（网络）
- **用途**: 监控网络请求，查看 API 调用情况
- **常用操作**:
  - 查看所有 HTTP 请求
  - 检查请求状态码（200 表示成功，404 表示未找到，500 表示服务器错误）
  - 查看请求和响应的详细内容
  - 过滤特定类型的请求（XHR、JS、CSS 等）

#### 3. Elements（元素）
- **用途**: 查看和修改 HTML 结构和 CSS 样式
- **常用操作**:
  - 检查页面元素的 DOM 结构
  - 实时修改样式进行调试
  - 查看元素的盒模型（margin、padding、border）

#### 4. Sources（源代码）
- **用途**: 调试 JavaScript 代码
- **常用操作**:
  - 设置断点进行调试
  - 单步执行代码
  - 查看变量值

#### 5. Application（应用）
- **用途**: 查看本地存储、Cookie、缓存等
- **常用操作**:
  - 查看 LocalStorage 中的数据
  - 清除缓存和存储数据

## 📊 如何查看错误信息

### 1. 浏览器控制台错误

**步骤**：
1. 打开开发者工具（F12）
2. 切换到 Console 标签页
3. 查找红色的错误信息

**常见错误类型**：
- **SyntaxError**: 语法错误，检查代码拼写
- **TypeError**: 类型错误，通常是访问了 undefined 或 null 的属性
- **ReferenceError**: 引用错误，使用了未定义的变量
- **Network Error**: 网络请求失败，检查 API 地址和网络连接

### 2. 终端错误

**位置**：运行 `npm run dev` 的终端窗口

**常见错误**：
- **编译错误**: TypeScript 或 ESLint 检查失败
- **模块未找到**: 缺少依赖包，需要运行 `npm install`
- **端口占用**: 需要更换端口或关闭占用进程

### 3. 网络请求错误

**步骤**：
1. 打开开发者工具的 Network 标签页
2. 刷新页面或触发相关操作
3. 查看失败的请求（红色显示）
4. 点击请求查看详细信息：
   - Headers: 请求头信息
   - Preview: 响应预览
   - Response: 完整响应内容

## 💡 开发技巧

### 快速重启

如果遇到问题，可以尝试完全重启开发环境：

```bash
# 1. 停止开发服务器（Ctrl+C）
# 2. 清除缓存并重新安装
rm -rf node_modules package-lock.json
npm install
# 3. 重新启动
npm run dev
```

### 查看项目构建

如果需要构建生产版本：

```bash
npm run build
```

构建完成后，可以预览生产版本：

```bash
npm run preview
```

### 代码检查

运行 ESLint 检查代码质量：

```bash
npm run lint
```
